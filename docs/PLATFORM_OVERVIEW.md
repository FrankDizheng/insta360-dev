# 大模型驱动的机械臂智能执行平台

> 版本：v1.1 | 2026-03-13

---

## 一、平台定位

本平台是一套 **"感知-决策-执行-学习" 闭环的机械臂智能基础设施**，以视觉大模型和动作大模型为核心引擎，驱动机械臂在不同场景下自主完成操作任务。

核心理念：**不为每个任务写死规则，而是让大模型在场景中自主构建规则、持续优化策略。**

四层架构：

- **场景层**：物料分拣、精密装配、质检搬运等，可灵活扩展
- **策略引擎**：视觉决策、动作生成、规则推理、技能库、异常处理
- **数据与学习层**：Case 自动采集、大模型分析、策略微调、模型微调
- **硬件抽象层**：机械臂控制、视觉传感器、力/力矩传感、边缘计算

---

## 二、要解决的问题

传统机械臂方案依赖人工编写固定规则和路径点，面临以下痛点：

| 痛点 | 传统方案 | 本平台方案 |
|------|----------|------------|
| 新场景部署周期长 | 每个场景重新编程，数周调试 | 自然语言描述任务，大模型自动构建逻辑 |
| 环境变化难适应 | 物体位置/姿态变化需重新标定 | 实时理解场景，动态调整决策 |
| 异常处理依赖人工 | 抓取失败需人工干预 | 闭环观察 + 自动修正 |
| 经验难以沉淀 | 调参经验在工程师个人手中 | Case 自动采集 → 大模型分析 → 技能库沉淀 |
| 迁移成本高 | A 场景的方案无法用于 B 场景 | 统一架构，新场景只需配置不需重开发 |
| 力控不灵活 | 固定力阈值，不同物体需反复调参 | 大模型根据物体材质/形状自适应输出力控策略 |

## 三、核心能力

### 3.1 视觉理解

大模型直接"看"摄像头画面，理解场景中有什么、在哪里、处于什么状态：

- **场景理解**：识别物体种类、材质、颜色、大小、空间关系
- **目标定位**：输出目标位置与姿态
- **状态判断**：判断抓取是否成功、物体是否到位、是否存在异常
- **2D→3D 映射**：结合深度相机将图像坐标转换为机械臂工作空间坐标

### 3.2 智能决策

| 能力 | 说明 |
|------|------|
| **任务规划** | 将自然语言指令（如"把蓝色瓶子放到左边箱子"）拆解为可执行的子任务序列 |
| **逐步决策** | 每一步拍照观察 → 判断当前状态 → 输出下一个动作 |
| **力控策略推理** | 根据目标物体的材质和形状，自动推理抓取力度、速度曲线和接近角度 |
| **计划复用** | 首次任务完整推理；成功后缓存为模板，后续同类任务直接复用 |
| **异常自修复** | 执行结果不符合预期时自动重新观察、调整策略 |

### 3.3 动作执行

端到端的动作大模型，直接从图像和语言指令生成关节级动作：

- 支持轻量级模型在边缘设备部署，满足实时性要求
- 支持从示教数据采集到模型微调的完整流程
- 不仅输出位置轨迹，还可输出各关节力矩参考值

### 3.4 自学习闭环

任务执行 → Case 采集 → 大模型分析 → 策略优化 → 技能库更新 → 下一轮任务

| 阶段 | 说明 |
|------|------|
| **采集** | 每次执行自动保存图像序列、动作日志、力传感数据、执行结果 |
| **分析** | 定期将失败/异常 case 批量提交给大模型分析 |
| **策略调整** | 根据分析结论更新决策规则、力控参数、识别阈值 |
| **模型微调** | 累积足够数据后对动作模型做 fine-tuning |
| **技能沉淀** | 成熟方案封装为 Skill，支持跨场景复用 |

## 四、系统架构

### 4.1 整体流程

自然语言指令 → 任务规划 → 视觉决策（结合相机+力传感器）→ 动作生成 → 机械臂执行 → 结果反馈 → 数据沉淀 → 定期分析 → 模型/策略迭代

### 4.2 核心设计

- **标准化服务接口**：AI 大脑与各子系统（视觉、机械臂、数据）之间通过标准化协议通信，模型可随时切换，不锁定任何供应商
- **多级加速策略**：通过智能缓存与分级推理机制，已识别的物料可在毫秒级完成决策，仅全新场景需要完整推理
- **硬件抽象层**：屏蔽底层差异，已适配 UR、JAKA、越疆等主流协作臂接口，新臂型接入周期 < 1 周

## 五、性能指标

### 5.1 响应时延

| 场景 | 端到端时延 | 说明 |
|------|-----------|------|
| 已识别物料（稳态） | **< 100 ms** | 智能缓存命中，约占日常 90%+ |
| 同类物料有变化 | **< 200 ms** | 相似度匹配后微调参数 |
| 全新物料/场景 | **3–5 s** | 完整推理，结果自动存入缓存 |

> 产线冷启动时每种新品类首次需完整推理（3–5s），之后自动缓存。品类稳定后日常运行平均决策延迟 **< 100ms**。

### 5.2 力控能力

| 指标 | 参数 |
|------|------|
| 力传感精度 | ±0.1 N |
| 力控频率 | 500–1000 Hz |
| 抓取力范围 | 0.5–50 N（覆盖鸡蛋到金属零件） |
| 碰撞检测 | < 5 ms 响应 |

**自适应力控策略** — 大模型根据物体材质自动匹配：

| 物体类型 | 抓取力 | 接近速度 | 示例 |
|----------|--------|----------|------|
| 脆弱/易碎 | 1–3 N | 慢 | 玻璃瓶、鸡蛋、芯片 |
| 普通/塑料 | 5–15 N | 中 | 塑料件、包装盒 |
| 坚硬/金属 | 15–40 N | 快 | 金属零件、工具 |
| 柔性/可变形 | 2–8 N | 慢 | 织物、线缆、软管 |

### 5.3 吞吐量

| 场景 | 节拍 |
|------|------|
| 简单分拣（缓存命中） | 3–5 s/件 |
| 常规抓放 | 6–10 s/件 |
| 复杂装配 | 15–30 s/步 |

### 5.4 定位精度

| 指标 | 参数 |
|------|------|
| 空间定位精度 | ±1–3 mm |
| 机械臂重复定位精度 | ±0.02–0.05 mm（工业级） |

## 六、场景自适应

### 6.1 新场景接入

1. **场景描述**（自然语言）— "从传送带上识别不同颜色的瓶子，分别放入对应的回收箱"
2. **自动拆解** — 识别颜色 → 确定目标箱 → 抓取 → 搬运 → 释放
3. **构建初始规则** — 拍照 → 理解场景 → 推理力控策略 → 首次执行计划
4. **首次执行 + 闭环观察** — 逐步执行，每步拍照 + 力反馈确认
5. **缓存 + 技能沉淀** — 成功方案固化，后续直接复用
6. **持续优化** — 失败 case 自动收集 → 大模型分析 → 微调策略

### 6.2 典型应用场景

| 场景 | 描述 | 关键能力 |
|------|------|----------|
| 物料分拣 | 按颜色/大小/类型将物料分入不同容器 | 视觉分类 + 快速节拍 |
| 精密装配 | 插接、对齐、拧紧等精密操作 | 力控柔顺 + 闭环修正 |
| 质检搬运 | 视觉检测缺陷 → 不良品分流 | 缺陷检测 + 分拣决策 |
| 柔性物体操作 | 线缆整理、织物折叠等 | 自适应力控 + 形变感知 |
| 人机协作 | 响应语音/手势指令协同作业 | 自然语言交互 + 安全力控 |
| 实验室自动化 | 移液、试管搬运、样本处理 | 高精度力控 + 防污染 |

## 七、典型问题：生产中物料形态变化怎么办？

传统方案依赖固定模板匹配，物料外形一变就得停线重新编程调参。本平台基于大模型语义理解，天然具备形态变化适应能力。

> **场景举例：** 生产线上的瓶子从圆瓶换成了方瓶，或者装箱的盒子尺寸变了。

| 方案 | 做法 | 耗时 |
|------|------|------|
| **传统方案** | 停线 → 工程师重新标定 → 调参 → 测试 → 恢复 | 数小时到数天 |
| **本平台** | 自动识别新形态 → 推理新策略 → 闭环确认 → 缓存 | 无需停线 |

### 不同变化程度的应对

| 变化程度 | 示例 | 停线时间 |
|----------|------|----------|
| 小变化 | 同类瓶子换尺寸（500ml→330ml） | **0** |
| 中等变化 | 换品类（圆瓶→方瓶） | **0** |
| 大变化 | 全新产品线上线 | **< 5 分钟** |

核心区别：传统方案依赖 **模板匹配**（模板变了就废了），本平台依赖 **语义理解**（理解的是"这是一个瓶子，要放进那个箱子"这个语义，不受具体外形约束）。
