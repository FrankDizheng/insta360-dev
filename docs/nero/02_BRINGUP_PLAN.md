# NERO Bring-up 计划

## 1. 官方已确认的信息

来自《NERO 用户手册》和官方开源资料：

- 通讯方式：CAN / HTTP / TCP
- 外部接口：电源接口、CAN 接口、网口
- 控制方式：拖动示教、离线轨迹、API、上位机
- 标准供电：DC24V
- 推荐二次开发入口：`pyAgxArm`
- 官方 ROS2 驱动：`agx_arm_ros`

其中 MVP 阶段优先走 CAN + Python SDK 或 ROS2 驱动，不建议一开始就依赖 Web 页面完成所有控制。

但按你们当前节奏，今天可以先不拉官方 GitHub 仓库、不绑定 SDK 实现，先把工程结构和接口抽象做完。

## 2. 到货当天的 Bring-up 顺序

### Step 1: 机械安装与供电检查

- 固定底座
- 清空工作空间
- 确认安装方式是正装、侧装还是倒装
- 确认供电电压不超过手册要求

注意：手册明确提到供电不可超过 25V，非标配供电也需要满足电流要求。

### Step 2: CAN 接线

- 机械臂航插接好
- USB 转 CAN 模块与 CAN_H / CAN_L 正确连接
- 笔记本识别 USB 转 CAN
- 激活 CAN 设备，波特率使用官方文档中的 1000000 bps

### Step 3: 网络侧验证

即使最终主控走 CAN，仍建议把 Web 后台和以太网先验证一遍，因为它对排查问题很有用。

- Wi-Fi 方式：确认能否登录 `192.168.31.1`
- 以太网方式：给笔记本配置同网段 IP，确认能通默认地址 `10.90.0.150`

### Step 4: 软件驱动验证

优先级顺序：

1. Python SDK `pyAgxArm`
2. ROS2 驱动 `agx_arm_ros`
3. Web 上位机辅助观察

原因：MVP 需要尽快进入 Python 工程集成，而不是长期停留在手工网页操作。

如果今天还不准备拉代码，这一步就先降级为：

1. 明确驱动接入接口
2. 预留 `NeroRobotController`
3. 用 mock 实现先打通主流程

## 3. 笔记本开发是否可行

可行，而且是当前最合理的第一阶段方案。

### 原因

- Python SDK 官方环境是 Ubuntu + Python，适合轻量开发机
- 机械臂控制不依赖本地大显卡
- 真正吃显存的是 VLM，不是 NERO 的执行控制
- 你们可以把 VLM 放在云端或远程 4090，笔记本只做调用方

### 结论

MVP 阶段不需要把主机搬进实验室。

推荐配置：

- 实验室笔记本：Ubuntu 优先，其次双系统 / WSL2 + Linux 外设链路
- 远程 GPU 机器：VLM 推理服务
- 局域网或公网 API：笔记本调用远程模型

## 4. 操作系统建议

### 最推荐

- Ubuntu 22.04 / 24.04 笔记本

理由：

- `pyAgxArm` 官方支持 Ubuntu 18.04 / 20.04 / 22.04 / 24.04
- `agx_arm_ros` 官方支持 ROS2 Humble / Jazzy
- CAN 工具链、ROS2、相机驱动都更顺手

### 当前可执行方案

- MacBook 先开发，远程部署 VLM，真机接入延后到硬件联调阶段

这条路线今天是成立的，因为你们当前先做的是：

- 抽象接口
- 服务编排
- 动作协议
- 视觉输入和日志链路

这些都不要求今天必须本地装好官方 SDK。

### 可接受

- Windows 笔记本 + Linux 远程开发环境

前提：

- CAN 接口和相机链路能稳定落在本机或虚拟机里
- 你们接受现场调试复杂度上升

如果是第一次 bring-up，我不建议把 Windows + 多层转发当主方案。

对于你们现在的节奏，我建议直接把 MacBook 当开发主机，等硬件到位后再决定是否补一台 Ubuntu 机器做现场控制。

## 5. 到货当天必须完成的验证项

- 机械臂能上电
- 状态灯正常
- CAN 能读到反馈
- 能获取关节状态
- 能使能 / 失能
- 能执行回零
- 能执行一条简单关节运动
- 能执行一条简单末端运动
- 如果有夹爪，能开合

只有这些都通过，视觉和 VLM 才值得接上来。

## 5.1 相机并行 Bring-up

既然相机是单独采购的，就要把它当成独立设备链路，而不是 NERO 附件。

相机侧到货后应单独验证：

- MacBook 能识别设备
- 能稳定抓单帧
- 能连续预览 10 到 15 分钟不掉线
- 能固定安装并覆盖完整工作区
- 目标物体在画面里的像素尺寸足够做识别与抓取点选择

如果是 RGB-D，相机侧再额外验证：

- 目标工作距离内的深度是否稳定
- 最小测距是否适合桌面抓取
- 近距离边缘噪声是否可接受

## 6. 到货当天不必做的事情

- 不急着做复杂视觉闭环
- 不急着跑仿真迁移
- 不急着做多物体任务规划
- 不急着上灵巧手复杂动作
- 不急着碰 MIT 模式

这些都会分散注意力。
