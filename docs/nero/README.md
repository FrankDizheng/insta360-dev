# NERO 集成开发文档

这组文档的目标不是介绍愿景，而是把 NERO 到货后的开发工作拆成可以直接执行的模块。

当前判断基于三类信息：

- 官方产品页与用户手册
- 官方 ROS2 驱动 `agx_arm_ros` 与 Python SDK `pyAgxArm`
- 本仓库现有的平台设计与可运行骨架代码

## 为什么现在就能开始

NERO 的控制侧并不要求本地一定有大 GPU。

- 机械臂控制本身走 CAN / HTTP / TCP，负载很轻
- 官方已提供 Python SDK 和 ROS2 驱动，适合用笔记本先把控制链路打通
- VLM 推理可以独立部署在云端或远程 4090 机器上，再由笔记本调用

所以当前最合理的开发方式是：

- 笔记本：现场 bring-up、控制、相机采集、服务编排、调试
- 远程算力：VLM 推理
- NERO：真机执行与状态反馈

## 关键边界

根据《NERO 用户手册》，标准装箱清单包含：

- 7 自由度机械臂
- USB 转 CAN 模块
- 电源适配器
- MicroUSB 线
- 固定螺丝与扳手

标准清单里没有写相机。

这意味着当前文档里的视觉层必须按“可插拔外设”设计，而不能假设标准 NERO 本体必带相机。如果你们到货的是带视觉套件的组合包，到货当天只需要把相机接入视觉层适配器，不需要改整体架构。

当前再补充一个实际约束：

- 相机正在单独采购
- 今天先用 MacBook 开发
- 今天先不依赖官方 SDK 仓库落地到本地

因此这组文档默认按以下方式推进：

- 先把平台接口、控制抽象、视觉抽象、VLM 服务协议写好
- 先用 mock 控制器和本机相机 / 测试图像把主流程跑通
- 等后续拿到官方代码或硬件到位，再把 NERO 真机控制实现接进来
