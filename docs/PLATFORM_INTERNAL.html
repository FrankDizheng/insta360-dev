大模型驱动的机械臂智能执行平台

技术方案概览 · v1.1 · 2026-03-13

一、平台定位

本平台是一套 "感知-决策-执行-学习" 闭环的机械臂智能基础设施，以 VLA（Vision-Language-Action）、VLM（Vision-Language Model）等大模型为核心引擎，驱动机械臂在不同场景下自主完成操作任务。

核心理念：不为每个任务写死规则，而是让大模型在场景中自主构建规则、持续优化策略。

二、要解决的问题

传统机械臂方案依赖人工编写固定规则和路径点，面临以下痛点：

痛点传统方案本平台方案
新场景部署周期长每个场景重新编程，数周调试自然语言描述任务，大模型自动构建逻辑
环境变化难适应物体位置/姿态变化需重新标定VLM 实时理解场景，动态调整决策
异常处理依赖人工抓取失败需人工干预闭环观察 + 自动修正

力控不灵活固定力阈值，不同物体需反复调参大模型根据物体材质/形状自适应输出力控策略

三、核心能力

3.1 视觉理解（VLM）

大模型直接"看"摄像头画面，理解场景中有什么、在哪里、处于什么状态：

  • 场景理解：识别物体种类、材质、颜色、大小、空间关系
  • 目标定位：Visual Grounding，输出目标 Bounding Box / 6DoF 位姿
  • 状态判断：判断抓取是否成功、物体是否到位、是否存在异常
  • 2D→3D 映射：结合 RGB-D 深度相机将图像坐标转换为机械臂工作空间坐标

3.2 智能决策（VLM + LLM）

能力说明
任务规划将自然语言指令（如"把蓝色瓶子放到左边箱子"）拆解为可执行的子任务序列
逐步决策每一步拍照观察 → VLM 判断当前状态 → 输出下一个动作
标准动作协议统一 JSON 格式：move_above / lower / grasp / lift / release / done
力控策略推理根据目标物体的材质和形状，自动推理抓取力度、速度曲线和接近角度
计划模板缓存首次任务由 VLM 逐步推理；成功后缓存为模板，后续同类任务跳过推理直接复用
异常自修复执行结果不符合预期时自动重新观察、调整策略

3.3 动作执行（VLA）

端到端的 Vision-Language-Action 模型，直接从图像和语言指令生成关节级动作：

  • SmolVLA：轻量级 VLA 模型，支持边缘设备部署
  • ACT：高精度模仿学习模型，一次预测未来 k 步轨迹
  • 训练管线：支持从示教数据采集到模型微调的完整流程，可对接工业级训练基础设施
  • 力矩输出：VLA 模型不仅输出位置轨迹，还可输出各关节力矩参考值

4.2 MCP 服务化架构

平台采用 MCP（Model Context Protocol） 作为 AI 大脑与各子系统的标准通信协议。每个能力模块封装为独立的 MCP Server，大模型通过统一的工具调用接口与之交互，实现模型可插拔、模块可替换。

MCP 协议的关键优势：模型可插拔。今天用 Qwen，明天换 GPT-4o 或 Claude，MCP Server 不用改动。对客户来说意味着不锁定任何一家 AI 供应商。

4.3 三级加速决策机制

为解决云端大模型调用延迟在生产线节拍中不可接受的问题，平台设计了 逐级 fallback 的加速架构：

模板精确匹配物料类型+场景 → 直接命中缓存
< 50ms
↓ 未命中
L2
向量相似匹配Embedding 相似度检索最近模板，微调参数
< 200ms
↓ 未命中
L3
边缘小模型推理Jetson 本地轻量 VLM（Qwen-VL-2B 量化）
200-500ms
↓ 信心不足
L4
云端大模型推理Qwen-Max / GPT-4o 完整推理 → 结果回写缓存
3-5s
生产线稳态运行时 90%+ 的请求命中 L1/L2，平均决策延迟 < 100ms。仅在首次遇到新品类或异常情况时才 fallback 到 L3/L4，且推理结果会自动回写缓存，后续不再触发。

七、典型问题：生产中物料形态变化怎么办？

传统方案依赖固定模板匹配，物料外形一变就得停线重新编程调参。本平台基于大模型语义理解，天然具备形态变化适应能力。

场景举例
生产线上的瓶子从圆瓶换成了方瓶，或者装箱的盒子尺寸变了。
❌ 传统方案
停线 → 工程师重新标定模板 → 调整抓取点和力控参数 → 反复测试 → 恢复生产
耗时数小时到数天
✅ 本平台
VLM 自动识别新物料形态 → 推理新抓取策略 → 首次闭环确认 → 缓存模板
全程无需人工干预

处理流程

瓶子形态变化
VLM 自动感知
"方形玻璃瓶，比之前的圆瓶更宽，表面平整"
推理新策略
平面夹持（非包裹式）、夹爪角度 0°、抓取力 4N
首次执行 + 闭环确认
~1-2s 推理 + 拍照验证
✓ 成功
缓存新模板，后续直接复用
✗ 失败
力反馈 → 自动调整策略 → 重试

不同变化程度的应对

变化程度示例处理方式停线时间
小变化同类瓶子换尺寸（500ml→330ml）VLM 自动适应，无需干预0
中等变化换品类（圆瓶→方瓶）VLM 首次推理建立新模板0
大变化全新产品线上线自然语言描述新规则，几次试抓验证< 5 分钟

为什么能做到？

核心区别：传统视觉方案依赖 模板匹配（模板变了就废了），本平台依赖 语义理解（VLM 理解的是"这是一个瓶子，要放进那个箱子"这个语义，不受具体外形约束）。

同理，容器/盒子变化也一样 —— VLM 会重新理解目标区域的位置、尺寸和开口方向，自动调整放置点和释放高度。
