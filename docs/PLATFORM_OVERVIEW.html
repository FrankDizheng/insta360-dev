大模型驱动的机械臂智能执行平台

技术方案概览 · v1.1 · 2026-03-13

一、平台定位

本平台是一套 "感知-决策-执行-学习" 闭环的机械臂智能基础设施，以视觉大模型和动作大模型为核心引擎，驱动机械臂在不同场景下自主完成操作任务。

核心理念：不为每个任务写死规则，而是让大模型在场景中自主构建规则、持续优化策略。

二、要解决的问题

传统机械臂方案依赖人工编写固定规则和路径点，面临以下痛点：

痛点传统方案本平台方案
新场景部署周期长每个场景重新编程，数周调试自然语言描述任务，大模型自动构建逻辑
环境变化难适应物体位置/姿态变化需重新标定实时理解场景，动态调整决策
异常处理依赖人工抓取失败需人工干预闭环观察 + 自动修正
经验难以沉淀调参经验在工程师个人手中Case 自动采集 → 大模型分析 → 技能库沉淀
迁移成本高A 场景的方案无法用于 B 场景统一架构，新场景只需配置不需重开发
力控不灵活固定力阈值，不同物体需反复调参大模型根据物体材质/形状自适应输出力控策略

三、核心能力

3.1 视觉理解

大模型直接"看"摄像头画面，理解场景中有什么、在哪里、处于什么状态：

  • 场景理解：识别物体种类、材质、颜色、大小、空间关系
  • 目标定位：输出目标位置与姿态
  • 状态判断：判断抓取是否成功、物体是否到位、是否存在异常
  • 2D→3D 映射：结合深度相机将图像坐标转换为机械臂工作空间坐标

3.2 智能决策

能力说明
任务规划将自然语言指令（如"把蓝色瓶子放到左边箱子"）拆解为可执行的子任务序列
逐步决策每一步拍照观察 → 判断当前状态 → 输出下一个动作
力控策略推理根据目标物体的材质和形状，自动推理抓取力度、速度曲线和接近角度
计划复用首次任务完整推理；成功后缓存为模板，后续同类任务直接复用
异常自修复执行结果不符合预期时自动重新观察、调整策略

3.3 动作执行

端到端的动作大模型，直接从图像和语言指令生成关节级动作：

  • 支持轻量级模型在边缘设备部署，满足实时性要求
  • 支持从示教数据采集到模型微调的完整流程
  • 不仅输出位置轨迹，还可输出各关节力矩参考值

七、典型问题：生产中物料形态变化怎么办？

传统方案依赖固定模板匹配，物料外形一变就得停线重新编程调参。本平台基于大模型语义理解，天然具备形态变化适应能力。

场景举例
生产线上的瓶子从圆瓶换成了方瓶，或者装箱的盒子尺寸变了。
❌ 传统方案
停线 → 工程师重新标定模板 → 调整抓取点和力控参数 → 反复测试 → 恢复生产
耗时数小时到数天
✅ 本平台
自动识别新物料形态 → 推理新抓取策略 → 首次闭环确认 → 缓存模板
全程无需人工干预

不同变化程度的应对

变化程度示例停线时间
小变化同类瓶子换尺寸（500ml→330ml）0
中等变化换品类（圆瓶→方瓶）0
大变化全新产品线上线< 5 分钟

核心区别：传统视觉方案依赖 模板匹配（模板变了就废了），本平台依赖 语义理解（理解的是"这是一个瓶子，要放进那个箱子"这个语义，不受具体外形约束）。
